# RC17 Attribution Policy

**Status:** Implementation attribution policy draft  
**Boundary:** Does not modify architecture, governance, ADRs, or the Open Grace canonical model.

## Purpose

Attribution protects source clarity for collection assets before publication. Every RC17 asset has an attribution record, even when publication is blocked.

## Attribution requirements

Every attribution record includes:

- attribution ID,
- asset ID,
- title,
- source ID,
- credit line,
- whether attribution is required,
- whether display is required.

## Display principle

If an asset is approved or later published, the publication surface should display source and license/status context in user-facing copy or metadata.

## Review principle

Attribution does not replace rights clearance. Assets with `Review Required` or `Restricted` rights status still require review before publication.
