# Nature & Culture Publication Approval Process

**Status:** RC17 implementation policy  
**Architecture note:** Architecture v1.0 remains frozen; this policy does not modify governance, architecture, agents, or ADRs.

## 1. Publication statuses

The publication approval registry uses:

- Draft
- Under Review
- Approved
- Rejected
- Published

## 2. Approval gates

An asset can move from Draft to Under Review when:

1. It has an asset registry record.
2. It has a source registry reference.
3. It has a provenance record.
4. It has an attribution record.
5. Its rights status is not Unknown.

An asset can move to Approved when:

1. Rights status is explicit.
2. Approval scope is defined.
3. Blocked uses are listed.
4. Review notes are recorded.

An asset can move to Published only when:

1. All rights/source details are complete for the intended use.
2. Jurisdiction rules are satisfied.
3. Attribution is display-ready.
4. Editorial approval is complete.

## 3. Demo approval

RC17 allows assets to be Approved for:

- public demo metadata
- editorial text reference
- placeholder product copy

This does not approve final commercial image use.

## 4. Prohibitions

This process does not enable payment processing, checkout, order creation, fulfillment, or customer data collection.
